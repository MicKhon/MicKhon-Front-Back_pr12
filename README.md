# 🔐 Практические занятия №7-12: Fullstack приложение с JWT-аутентификацией и RBAC
# Полнофункциональное веб-приложение с React.js фронтендом и Node.js бэкендом, реализующее многоуровневую систему безопасности с JWT-аутентификацией, refresh-токенами и управлением доступом на основе ролей (RBAC).

## Практика №7: JWT-аутентификация
### 📖 Описание технологии:
#### JSON Web Token (JWT) — это стандарт RFC 7519 для безопасной передачи информации между сторонами в виде JSON-объекта. JWT состоит из трех частей:
1) Header (заголовок) — тип токена и алгоритм шифрования
2) Payload (полезная нагрузка) — данные пользователя
3) Signature (подпись) — проверка целостности токена

### Что реализовано:
1) Регистрация и вход пользователей
2) Генерация JWT-токенов
3) Защита маршрутов middleware
4) Передача токена в заголовке Authorization

## Практика №8: Хеширование паролей и middleware
### 📖 Описание технологии:
#### bcrypt — библиотека для хеширования паролей с использованием алгоритма bcrypt. Основные преимущества:
1) Соль (salt) — случайные данные, добавляемые к паролю перед хешированием
2) Раунды хеширования — количество повторений алгоритма (в проекте: 10 раундов)
3) Необратимость — невозможно восстановить пароль из хеша
4) Middleware — функции, которые выполняются до обработки запроса основными обработчиками маршрутов.

### Что реализовано:
1) Безопасное хранение паролей (хеширование)
2) Проверка паролей при входе
3) Логирование всех запросов
4) Обработка ошибок в middleware

## Практика №9: Refresh-токены
### 📖 Описание технологии
#### Refresh-токены — долгоживущие токены для обновления access-токенов без повторного ввода учетных данных. Решают проблему короткого времени жизни access-токенов:
1) Access Token — короткоживущий (15 минут), для доступа к API
2) Refresh Token — долгоживущий (7 дней), для получения новой пары токенов

### Что реализовано:
1) Генерация пары access + refresh токенов
2) Хранилище refresh-токенов (Set)
3) Маршрут POST /api/auth/refresh
4) Ротация токенов (старый удаляется, новый создается)

---

## Практика №10: Хранение токенов на фронтенде
### 📖 Описание технологии
#### Способы хранения токенов:
1) localStorage — простое хранилище ключ-значение, доступно через JavaScript
2) HttpOnly Cookies — cookies, недоступные для JavaScript (защита от XSS)
3) Axios Interceptors — перехватчики запросов и ответов для автоматизации работы с токенами

### Что реализовано:
1) Access-токен в localStorage
2) Refresh-токен в HttpOnly cookie
3) Axios interceptors для автоматической подстановки токена
4) Автоматическое обновление токена при 401 ошибке
5) Redirect на /login при истечении refresh-токена

## Практика №11: RBAC (Role-Based Access Control)
### 📖 Описание технологии
#### RBAC (Role-Based Access Control) — модель контроля доступа на основе ролей. Права доступа определяются не для каждого пользователя отдельно, а для ролей:
1) Guest (Гость) — неаутентифицированный пользователь
2) User (Пользователь) — аутентифицированный пользователь (просмотр)
3) Seller (Продавец) — сотрудник (добавление и редактирование товаров)
4) Admin (Администратор) — управленец (полные права + управление пользователями)

### Что реализовано:
1) Система ролей (user, seller, admin)
2) roleMiddleware для проверки ролей
3) Маршруты /api/users доступны только админу
4) Создание/редактирование товаров доступно seller и admin
5) Удаление товаров доступно только admin
6) Фронтенд адаптирован под RBAC
7) Панель администратора (/admin)
8) Компонент тестового входа
